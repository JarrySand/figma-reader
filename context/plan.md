# DAO大学 構想具体化プラン

## 目的

3/15合宿で作成したステークホルダーマップ（`output/ai-rearranged.md`）を起点に、DAO大学の構想を実行可能なレベルまで具体化する。

---

## フェーズ① ペルソナ設計

ステークホルダーマップの22カテゴリ＋個別ノードを、以下の3つの役割軸で整理し、具体的なペルソナに落とし込む。

### 役割分類

| 役割           | 定義                                                  | 例                                       |
| -------------- | ----------------------------------------------------- | ---------------------------------------- |
| **直接受益者** | DAO大学の講座を直接受講する人                         | 起業志望の学生、DAO事業をやりたい社会人  |
| **間接受益者** | DAO大学の活動によって間接的に価値を受ける／期待する人 | 自治体、地域コミュニティ、企業の人事部門 |
| **負担者**     | 意義を感じてカネ・モノ・リソースを提供してくれる人    | 富裕層、地銀、CSR企業、クリプトファンド  |

> **注意点：**
>
> - 受益者兼負担者（例：CSR企業＝社会貢献アピール＋資金提供、富裕層＝資産活用＋寄付）が存在する
> - 受益だけを受ける人（例：未来の子供、自然、被災地、寝たきりの高齢者）も存在する
> - 一人のペルソナが時間軸で役割を移行する場合がある（受講者→講師、受益者→負担者）

### ペルソナに含める要素

- 名前・年齢・属性（架空の代表人物）
- 現在の課題・ペインポイント
- DAO大学に期待すること（ゲイン）
- 役割分類（直接受益者／間接受益者／負担者／兼任）
- 接点チャネル（どこで出会うか）
- 提供できるもの（負担者の場合）

## フェーズ② カスタマージャーニー作成

各ペルソナについて、DAO大学との接点から行動変容までのジャーニーを設計する。

### ジャーニーの段階

```
認知 → 興味 → 参加（受講/支援） → 体験 → 変容 → 伝播/貢献
```

> エコシステム循環モデル（3/15合宿で設計）と対応させる：
>
> - 「伝播」= 受ける人が他者に広める（UGC的拡散）
> - 「貢献」= 受ける人が提供する人・受益者へ進化する

### ジャーニーに含める要素

- 各段階でのタッチポイント
- 各段階での感情・思考
- 各段階でDAO大学が提供するもの
- 離脱リスクと対策
- 次の段階への転換トリガー

### 作成順序

1. **コアペルソナ**（直接受益者の代表3〜4名）を先行
2. **負担者ペルソナ**（資金・リソース提供者の代表2〜3名）
3. **間接受益者ペルソナ**（自治体・地域等の代表2〜3名）

## フェーズ③ 実行ロードマップ

ペルソナとジャーニーをもとに、DAO大学の立ち上げから運営までのロードマップを作成する。

### ロードマップの構成要素

- フェーズ分け（準備期 → パイロット期 → 拡大期）
- 各フェーズのKPI
- 必要なリソース（人・金・ツール）
- 講座コンテンツの優先順位
- 認定制度の設計
- トークン/ポイント設計
- フリーライド対策の具体施策
