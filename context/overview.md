# 日本DAO協会（JDA）戦略 Overview

## プロジェクト概要

日本DAO協会の活動方針・戦略を策定するプロジェクト。2026年2月〜3月にかけて計3回の戦略会議を実施し、協会のミッション定義からDAO大学構想の具体化まで議論が進展した。

## 会議の経緯と進展

### 第1回：2/24 — DAOが浸透しない理由の分析と役割分担

**核心的な問い：** なぜDAOは浸透しないのか？

**結論：** 「やる理由が弱く、やるコストが高い」という構造的問題

- **やる理由の弱さ：** 経済的価値（収益性・資金調達・コスト削減）と社会的価値（ソーシャルインパクト・既存組織との差別化）が不足
- **やるコストの高さ：** 技術的専門性、人材不足、理解・合意形成の困難さ、制度未整備によるリスク

**民間企業とDAO協会の役割分担：**
- **民間企業** → 収益性があり模倣困難な領域（コミュニティ、ネットワーク効果など）
- **DAO協会** → コモディティ化しやすい基礎的部分（テンプレート、ツール、基礎教育、政策提言）

**アクション：** 関係者の利害関係を整理する「相利評価票」の作成を決定

### 第2回：3/4 — ステークホルダーの相利評価とDAO大学の決定

**協力3.0の概念：** 全員が異なる目的を持ちながらも、共同活動を通じて各自の利益を得られる仕組み

**DAO協会のミッション：** 「DAOプレーヤーの挑戦と実験のコストを下げること」

**協会の活動領域（3分類）：**
1. **公共財的領域** — 定款・規定作成
2. **収益性はあるが標準化しやすい領域** — スマートコントラクト、テンプレート類、基礎教育
3. **直接収益にはならないが戦略的価値を持つ領域** — 政策提言、業界内ポジション取得

**事業者間の棲み分け合意：**
- さわくん → 地域全体のDAO設計
- 島村氏（貴凛庁） → エコシステムデザイン
- Aram Mine氏（ガイアックス） → 事業型DAOと資金調達
- Uwaizumi.eth氏（Unyte） → 評価システム

**DAO大学の決定：** macの提案により設立が合意。講座形式でDAOの知識を提供し、各事業者が講師として参加。基礎知識はオープン化、専門コンサルは各事業者が担当。

---

### 第3回：3/15 — DAO大学エコシステム構想の戦略ブレスト（合宿＠キボッチャ）

**エコシステムの3つの役割：**
1. **提供する人** — 講師・事業者として知識やリソースを提供
2. **受ける人** — 講座を受講する人（＝市場、toCに近い）
3. **受益者** — 間接的に恩恵を受ける人（行政、企業、コミュニティなど、toBに近い）

**循環モデルの設計（線形→ネットワーク効果）：**
- 受ける人が単に知識を受け取るだけでなく、他者に伝播（UGC的拡散）し、やがて提供する人・受益者へと進化する循環が不可欠
- 労働集約型の線形モデルを避け、ストック（蓄積されるアセット＝ナレッジ・データ・ネットワーク）を持つ設計が必要
- **参考モデル：** Amazonダブルフライホイール — マッチングを通じたデータ蓄積→自社商品開発

**協力モデルの類型：**
- **協力1.0** — 全員が同じ目的を共有（林業モデル）
- **協力2.0** — サプライチェーン型、利害が対立しない
- **協力3.0** — 異なる目的のステークホルダーが相利関係を構築（DAO協会はこれ）

**受益者・負担者の具体候補：**
- 短期優先：自治体、大企業、地方銀行、遺産活用を考える富裕層（お金を払える層）
- 長期的持続性：学生・インターン（活きのいい若者）
- 「DAO大学奨学金」のようなフォーマットで支援者の受け皿を用意

**DAO大学の二本柱：**
1. **講義** — 知識の体系的な提供
2. **フィールドワーク** — 実践的な学び

**フリーライド対策：** トークンベースの権限管理、DAOガバナンスで対処

**認定制度：** 提供者（講師・教授）に対する認定制度の導入を合意

## 現在のステータス

DAO大学エコシステムの循環モデルが概念設計され、次のステップとして「受ける人に提供する具体的なコンテンツ」のブレインストーミングに進む段階。
